# Практическая работа: слияние веток и разрешение конфликтов

Этот репозиторий создан для отработки навыков слияния веток и разрешения конфликтов в Git.

## Задание213123

### Часть 1. Подготовка

1. Убедитесь, что Git установлен и настроен (имя и email). Если нет — выполните настройку:
   ```
   git config --global user.name "Ваше Имя"
   git config --global user.email "ваш_email@example.com"
   ```

2. Сделайте **fork** этого репозитория на свою учётную запись GitHub.

3. Склонируйте свой fork на локальный компьютер:
   ```
   git clone https://github.com/ваш_логин/название_репозитория.git
   ```

4. Перейдите в папку репозитория:
   ```
   cd название_репозитория
   ```

### Часть 2. Создание своей ветки и первого коммита

5. Создайте новую ветку с названием `feature-ваша_фамилия` (например, `feature-ivanov`):
   ```
   git checkout -b feature-ivanov
   ```

6. В этой ветке создайте папку со своей фамилией (кириллицей). Внутри папки создайте текстовый файл `review.txt` и напишите в нём пару предложений о том, что вам понравилось или показалось сложным в Git.

7. Добавьте файл в индекс и сделайте коммит:
   ```
   git add фамилия/review.txt
   git commit -m "Добавил отзыв от [ваша фамилия]"
   ```

### Часть 3. Создание конфликтной ситуации

8. Переключитесь на ветку `main`:
   ```
   git checkout main
   ```

9. Находясь в ветке `main`, создайте файл `students.txt` и запишите в него:
   ```
   Текущий список студентов:
   [ваша фамилия] (основная ветка)
   ```
   Сохраните файл.

10. Закоммитьте этот файл в ветке `main`:
    ```
    git add students.txt
    git commit -m "Добавил students.txt в main"
    ```

11. Вернитесь в свою ветку `feature-ваша_фамилия`:
    ```
    git checkout feature-ivanov
    ```

12. В этой ветке **также** создайте файл `students.txt` и запишите в него:
    ```
    Студенты, прошедшие курс:
    [ваша фамилия] (моя ветка)
    ```
    Сохраните файл.

13. Закоммитьте этот файл в своей ветке:
    ```
    git add students.txt
    git commit -m "Добавил students.txt в feature-ветку"
    ```

### Часть 4. Слияние и разрешение конфликта

14. Выполните слияние ветки `main` в вашу текущую ветку:
    ```
    git merge main
    ```

    Git сообщит о конфликте в файле `students.txt`.

15. Откройте файл `students.txt` в любом текстовом редакторе. Вы увидите маркеры конфликта:
    ```
    <<<<<<< HEAD
    Студенты, прошедшие курс:
    [ваша фамилия] (моя ветка)
    =======
    Текущий список студентов:
    [ваша фамилия] (основная ветка)
    >>>>>>> main
    ```

16. Отредактируйте файл так, чтобы объединить оба варианта или выбрать один. Например:
    ```
    Студенты, прошедшие курс:
    [ваша фамилия] (моя ветка)
    Текущий список студентов:
    [ваша фамилия] (основная ветка)
    ```
    Удалите маркеры `<<<<<<<`, `=======`, `>>>>>>>`.

17. Сохраните файл и добавьте его в индекс (это покажет Git, что конфликт разрешён):
    ```
    git add students.txt
    ```

18. Завершите слияние коммитом:
    ```
    git commit
    ```
    Git автоматически откроет редактор с сообщением о слиянии. Вы можете оставить предложенное сообщение или написать своё.

### Часть 5. Отправка на GitHub и Pull Request

19. Отправьте свою ветку на GitHub:
    ```
    git push origin feature-ваша_фамилия
    ```
    (Если ветка ещё не была отправлена, Git подскажет более длинную команду с флагом `-u`.)

20. Зайдите на GitHub в свой fork и создайте **Pull Request** из ветки `feature-ваша_фамилия` в основную ветку **исходного репозитория** (того, от которого вы делали fork).

21. В описании Pull Request кратко напишите, с каким конфликтом вы столкнулись и как его решили.

## Полезные команды (шпаргалка)

| Команда | Что делает |
|---------|------------|
| `git status` | Показывает текущее состояние репозитория |
| `git log --oneline --graph` | Показывает историю коммитов в виде графа |
| `git branch` | Показывает список веток |
| `git checkout -b <имя>` | Создаёт и переключается на новую ветку |
| `git add <файл>` | Добавляет файл в staging |
| `git commit -m "сообщение"` | Создаёт коммит |
| `git merge <ветка>` | Сливает указанную ветку в текущую |
| `git push origin <ветка>` | Отправляет ветку на GitHub |
